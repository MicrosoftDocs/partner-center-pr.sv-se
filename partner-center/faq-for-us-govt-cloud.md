# <a name="frequently-asked-questions-for-partner-center-for-microsoft-cloud-for-us-government"></a>Vanliga frågor och svar om Partnercenter för Microsoft Cloud for US Government

**Gäller för**: PartnerCenter för Microsoft Cloud for US Government

**Lämpliga roller:** Global administratör

## <a name="general-questions"></a>Allmänna frågor

Använd dessa vanliga frågor och svar om du vill veta mer om programmet Molnlösningsleverantör (CSP) för Microsoft Cloud for US Government. Du kan också kontakta din kontoansvariga om du vill ha mer information.

### <a name="what-is-microsoft-cloud-for-us-government"></a>Vad är Microsoft Cloud for US Government?

Den har utformats för amerikanska myndigheter från grunden och Microsoft Cloud for US Government gör det möjligt för kunder inom den offentliga sektorn i USA – från stora federala myndigheter till små myndigheter – att välja bland en mängd olika tjänster för molnbaserad databehandling. För att uppfylla kundernas specifika behov har Microsoft gjort betydande investeringar i datacenter och är dedikerade för att uppfylla efterlevnad med amerikanska federala och statliga principer, krav och krav.

### <a name="who-is-eligible-for-the-csp-program-for-microsoft-cloud-for-us-government"></a>Vem är berättigad till CSP-programmet för Microsoft Cloud for US Government?

Partner i USA som betjänar amerikanska federala, statliga och lokala myndigheter i deras myndighetsorganisationer är berättigade till CSP-programmet för Microsoft Cloud for US Government. Microsoft har ett strikt valideringsprogram för att fastställa berättigande innan partner kan komma åt Microsoft Cloud for US Government. För att kvalificera dig måste du bevisa att du arbetar med myndighetskunder. Registreringsprocessen för CSP-programmet i Microsoft Could for US Government har förstärkts för att samla in mer data för att utföra verifieringen. Typer av bevis kan inkludera, men är inte begränsade till, myndighetskontraktnummer och sponsringsbrev från din myndighetskund.

### <a name="is-customer-pricing-the-same-as-for-csp-commercial"></a>Är kundpriser samma som för CSP Commercial?

Nej. Prissättningen återspeglar de ytterligare kostnaderna för att ge det unika värdet Microsoft Cloud for US Government.

### <a name="is-the-partner-discounting-model-the-same"></a>Är partnerns rabattmodell densamma?

Ja. Partnerrabattmodellen för CSP-programmet för Microsoft Cloud for US Government är samma som partnerrabattmodellen för CSP Commercial.

### <a name="csp-commercial-is-an-existing-partner-program-how-is-the-csp-program-for-microsoft-government-cloud-different"></a>CSP Commercial är ett befintligt partnerprogram. Hur skiljer sig CSP-programmet för Microsoft Government-molnet åt?

CSP-programmet för Microsoft Cloud for US Government skiljer sig från CSP Commercial inom följande områden:

- Berättigande: CSP-programmet för Microsoft Cloud for US Government är endast tillgängligt för partner i USA som betjänar amerikanska federala, statliga och lokala myndigheter i deras myndighetsorganisationer. Microsoft har ett valideringsprogram för att fastställa berättigande innan partner kan komma åt us government-molnet.

- Registrering: Partner måste registrera till CSP-programmet för att Microsoft Cloud for US Government från CSP Commercial.

- Produkttillgänglighet: Både Azure Government och licensbaserade tjänster som Office 365, Enterprise Mobility + Security och Dynamics 365 är tillgängliga i CSP-programmet för Microsoft Cloud for US Government.

- Kontrakt: McRA (Microsoft Cloud Reseller Agreement) har uppdaterats för att återspegla unika myndighetsvillkor.

- PartnerCenter-funktioner: Vissa funktioner i Partnercenter aktiveras inte för CSP-programmet för Microsoft Cloud for US Government att uppfylla regel- och efterlevnadskrav. Partner kan i stället använda Azure Active Directory (Azure AD) [eller Graph API](/partner-center/develop/developing-for-partner-center-for-microsoft-national-cloud#partner_center_msftcloudUS) att utföra uppgifter som rör användar- och grupphantering. Den automatiserade funktionen Relationsbegäran är avsedd att göras tillgänglig i oktober 2021.

### <a name="i-am-not-a-csp-partner-yet-where-can-i-find-more-information-on-how-to-become-a-csp-partner"></a>Jag är inte CSP-partner ännu. Var hittar jag mer information om hur jag blir CSP-partner?

Mer information om registreringsmodeller som är tillgängliga för partner [finns Molnlösningsleverantör på Microsoft Partner Network](https://partner.microsoft.com/cloud-solution-provider).

### <a name="how-does-licensing-work"></a>Hur fungerar det med licensiering?

*Microsoft Cloud Reseller Agreement (MCRA)*

MCRA ger partner tillstånd att sälja med CSP. US Government Cloud-versionen av det här avtalet godkänns vid registrering till Microsoft Cloud for US Government. Den här versionen har ett tillägg för US Government Cloud och uppvisar A efter standardavtalet för molnåterförsäljare.

MCRA för Microsoft Cloud for US Government skiljer sig från MCRA-standard på följande två sätt:

- Us Government Cloud Addendum innehåller ytterligare partneransvar för att göra en transacting av US Government Cloud-tjänster.

- Bilaga A innehåller Microsoft-kundavtal (Microsoft-kundavtal) som måste införlivas i varje kontrakt för US Government Cloud-tjänster som ingår mellan molntjänsttjänsten och dess kund.

*Microsoft-kundavtal (Microsoft-kundavtal) som visas som bilaga A i MCRA*

CSP:er måste införliva bilaga A i myndighetskontraktet som anges mellan CPS och kunder som köper Microsoft Cloud for US Government. Avtalet publiceras i Partnercenter och är tillgängligt på engelska.

Microsoft-kundavtal för US Government Cloud skiljer sig från Microsoft-kundavtal på följande två sätt:

- Det måste införlivas i CSP:s kontrakt med kunden.

- Det har skräddarsytts för att uppfylla federala, statliga och lokala lagar och föreskrifter.

*Microsoft Cloud Distributor Agreement (MCDA)*

Microsoft Cloud Distributor Agreement (MCDA) tillåter distributörer att sälja via CSP. US Government Cloud-versionen av MCDA har ett tillägg för US Government Cloud och uppvisar A efter Standard Microsoft Cloud Reseller Agreement (MCRA).

MCDA för Microsoft Cloud for US Government skiljer sig från MCDA-standard på följande två sätt:

- Microsoft Cloud for US Government-tillägget innehåller ytterligare partneransvar för att göra en transacting av US Government Cloud-tjänster.

- Visa A innehåller Microsoft-kundavtal (Microsoft-kundavtal) som måste införlivas i varje kontrakt för US Government Cloud-tjänster som ingår mellan CSP:n och dess kunder.

### <a name="what-are-the-qualifications-to-sell-microsoft-cloud-for-us-government"></a>Vilka är kvalificeringarna för att sälja Microsoft Cloud for US Government?

Kvalificeringskriterierna för att sälja CSP-programmet för Microsoft Cloud for US Government är samma som kvalificeringskriterierna för CSP Commercial, som är öppna för både direkta partner och indirekta återförsäljare. Du behöver inte vara en indirekt återförsäljare för CSP-programmet för Microsoft Cloud for US Government om du är godkänd som en indirekt återförsäljare för CSP Commercial.

### <a name="i-am-an-existing-partner-what-do-i-need-to-do-to-sell-microsoft-cloud-for-us-government"></a>Jag är en befintlig partner. Vad behöver jag göra för att sälja Microsoft Cloud for US Government?

Partner måste registrera sig för CSP-programmet för att Microsoft Cloud for US Government uppfylla regel- och efterlevnadskrav. (Onboarding för Microsoft Cloud for US Government är separat från onboarding för CSP Commercial.) När du har introducerat kan du skapa kunder och transact-erbjudanden som är tillgängliga inom Microsoft Cloud for US Government. Om du använder API:er kan du gå [till Utveckla för Partner Center för Microsoft National Cloud.](/partner-center/develop/developing-for-partner-center-for-microsoft-national-cloud#partner_center_msftcloudUS)

### <a name="is-there-any-difference-in-partner-incentives"></a>Finns det någon skillnad i partnerincitament?

Nej. Partnerincitament för CSP-programmet för Microsoft Cloud for US Government är samma som partnerincitament för CSP Commercial. Om du vill veta mer om partnerincitament loggar du in [på Microsoft Partner Network portalen.](https://partner.microsoft.com/membership/partner-incentives)

### <a name="are-there-any-changes-to-billing"></a>Finns det några ändringar i faktureringen?

Nej. Faktureringsvillkor och krav för CSP-programmet för Microsoft Cloud for US Government är samma som faktureringsvillkor och krav för CSP Commercial. Mer information om fakturering finns i resurserna på [Microsoft Partner Network](https://partner.microsoft.com/cloud-solution-provider/resources).

### <a name="for-data-hosted-in-commercial-data-centers-is-it-possible-to-change-to-microsoft-cloud-for-us-government"></a>Är det möjligt att ändra till att lagra data i kommersiella datacenter Microsoft Cloud for US Government?

För att underlätta och upprätthålla integriteten hos isoleringen av Microsoft Cloud for US Government finns det inga mekanismer för att flytta en kunds befintliga tjänster till Microsoft Cloud for US Government. Om en kund eller partner har Azure ansvarar de för alla data som flyttas till en ny miljö inom Microsoft Cloud for US Government.

## <a name="questions-about-microsoft-cloud-for-us-government-for-azure"></a>Frågor om Microsoft Cloud for US Government för Azure

### <a name="what-is-microsoft-cloud-for-us-government-for-azure"></a>Vad är Microsoft Cloud for US Government för Azure?

Azure Government moln levererar en molnplattform som bygger på grundläggande principer för säkerhet, sekretess och kontroll, efterlevnad och transparens. Entiteter inom den offentliga sektorn får en fysiskt isolerad instans av Microsoft Azure som använder förstklassiga säkerhets- och efterlevnadstjänster som är viktiga för amerikanska myndigheter för alla system och program som bygger på dess arkitektur. Dessa tjänster omfattar FedRAMP- och DoD-efterlevnadscertifieringar, CJIS-tillståndsavtal, möjligheten att utfärda HIPAA-affärsavtal och stöd för IRS 1075. Drivs av säkerhetsbekanta amerikanska Azure Government stöd för flera hybridscenarier för att skapa och distribuera lösningar lokalt eller i molnet. Entiteter inom den offentliga sektorn kan också dra nytta av omedelbar skalbarhet och garanterad drifttid för en molntjänst i hyperskala.

### <a name="does-the-csp-program-for-microsoft-cloud-for-us-government-offer-all-azure-services"></a>Erbjuder CSP-programmet för Microsoft Cloud for US Government alla Azure-tjänster?

Infrastruktur som en tjänst (IaaS) och grundläggande plattforms- och dataerbjudanden finns nu tillgängliga. Översikten uppdateras kontinuerligt och vi tillhandahåller dokumentation som liknar dokumentationen för CSP Commercial i Partner Center (autentisering krävs). Du kan granska Azure Government Services som för närvarande är tillgängliga här, men observera att vissa tjänster, till exempel klassisk beräkning, nätverk, lagring och molntjänster inte är utformade för och fungerar inte för CSP.

### <a name="what-do-i-need-to-do-to-develop-with-the-csp-program-for-microsoft-cloud-for-us-government"></a>Vad behöver jag göra för att utveckla med CSP-programmet för Microsoft Cloud for US Government?

Mer information om hur du utvecklar med PartnerCenter för Microsoft Cloud for US Government finns i [Utveckla med Partner Center för Microsoft Government Cloud](/partner-center/develop/developing-for-partner-center-for-microsoft-national-cloud#partner_center_msftcloudUS).

### <a name="are-there-any-differences-in-apis"></a>Finns det några skillnader i API:er?

API:er och integrationskrav för CSP-programmet för Microsoft Cloud for US Government är identiska med API:er och integrationskrav för CSP Commercial. Mer information om slutpunkter i Microsoft Cloud for US Government finns i [Rest-URL:er för PartnerCenter.](/partner-center/develop/partner-center-rest-urls)

>[!NOTE]
>POR-API:er kommer inte att vara tillgängliga för CSP-programmet för Microsoft Cloud for US Government. Dessa API:er är inaktuella i CSP och är därför inte tillgängliga för Microsoft Cloud for US Government. För att uppfylla regel- och efterlevnadskrav är följande API:er inte tillgängliga i CSP-programmet för Microsoft Cloud for US Government.
>
> **Hantering av användarkonto**
>- Skapa användarkonton för en kund
>- Ta bort användarkonton för en kund
>- Uppdatera användarkonton för en kund
>- Hämta en lista över alla användarkonton för en kund
>- Återställ användarlösenord för en kund
>- Hämta användarroller för en kund
>- Ange användarroller för en kund
>- Visa borttagna användare för en kund
>- Återställ en borttagen användare för en kund
>
> **Licenshantering**
>- Hämta en lista över tillgängliga licenser
>- Tilldela licenser till en användare
>-  Kontrollera vilka licenser som har tilldelats till en användare

### <a name="is-there-a-workaround-for-apis-not-available-in-the-csp-program-for-microsoft-cloud-for-us-government"></a>Finns det någon lösning för API:er som inte är tillgängliga i CSP-programmet för Microsoft Cloud for US Government?

Partner kan använda Azure Portal, Azure AD PowerShell eller Graph API för hantering av partneranvändarkonto. Mer teknisk information finns i [Utveckla med Partner Center for Government Cloud](https://msdn.microsoft.com/partner-center/partner-center-for-microsoft-us-govt-cloud).
